# Фонд оценочных средств по дисциплине «Инженерия данных для систем искусственного интеллекта»

Репозиторий содержит фонд оценочных средств (ФОС) по дисциплине «Инженерия данных для систем искусственного интеллекта»: рабочую программу, модель измерения результатов обучения, контрольно-измерительные материалы (КИМ), критерии оценивания, методические указания и информационные ресурсы. Результаты обучения связаны с компетентностно-ролевой моделью в области искусственного интеллекта (КРМ версии 3.0).

## 1. О дисциплине

**Цель дисциплины.** Научить студента превращать разнородные и неструктурированные данные в надёжную инфраструктуру для аналитики и ИИ-продуктов и подготовить его к работе в роли инженера данных (Data Engineer) в командах, разрабатывающих ИИ-системы.

**Основные задачи.**
1. Сформировать понимание архитектур систем хранения данных: источник → сбор и интеграция → очистка и валидация → КХД / озеро данных → витрина → аналитический / ИИ-контур потребления.
2. Сформировать навыки классификации источников данных и методов их сбора — структурированных, неструктурированных, потоковых, IoT, графовых и документных.
3. Обеспечить владение подходами проектирования корпоративных хранилищ: Kimball, Inmon, Data Vault, а также концепциями DWH, Data Lake, Data Lakehouse, Data Fabric и Data Mesh.
4. Сформировать умения проектировать архитектуры хранения для ИИ — Feature Store, векторные БД, векторные, графовые и гибридные хранилища для RAG.
5. Обеспечить понимание интеграционных шин, ETL/ELT и облачных инструментов их построения.
6. Сформировать навыки оценки качества данных (полнота, непротиворечивость, актуальность, достоверность) и методов очистки, нормализации и валидации.
7. Сформировать навыки проектирования и реализации ETL/ELT-конвейеров для отраслевых кейсов с оценкой корректности, производительности и надёжности.

**Структура.** Дисциплина включает 10 модулей:

1. Введение в архитектуру данных для ИИ.
2. Источники данных и методы их сбора.
3. Модели хранилищ данных.
4. Концепции корпоративных хранилищ данных.
5. Современные архитектуры хранилищ для ИИ и MLOps.
6. Векторные, графовые, гибридные хранилища для RAG-систем.
7. Интеграционные технологии.
8. Подготовка и качество данных.
9. Проектирование ETL/ELT-конвейеров.
10. Облачные инструменты построения ETL/ELT-конвейеров и КХД.

Формы работы: лекции, лабораторные занятия, самостоятельная работа. Промежуточная аттестация — экзамен.

**Пререквизиты.**

| Дисциплина | Компетенция КРМ / направление | Входной уровень |
|---|---|---|
| Алгоритмы и структуры данных | Языки программирования (фундамент) | Б |
| Языки программирования | PL-1. Python / PL-2. Java-Scala-Kotlin | Б–П |
| SQL, базы данных (реляц./нереляц.) | BD-3. Хранение данных | Б |
| Основы ML | ML-2. Основы машинного обучения | Б |
| Проектирование API | LC-3. Проектирование систем ИИ (ML System Design) | Б |
| Проектирование АИС (жизненный цикл) | LC-1 / LC-4 | Б |
| ИТ-инфраструктура, виртуализация, мониторинг | LC-5 | Б |
| CI/CD | LC-5 | Б |

**Результаты обучения.** По завершении дисциплины обучающийся сможет:
- осознанно выбирать архитектуру под отраслевой кейс;
- работать с разнородными источниками данных;
- обоснованно проектировать корпоративное хранилище данных;
- готовить данные для MLOps- и RAG-систем;
- выбирать и настраивать инструменты интеграции в облачной среде;
- предотвращать ошибки ИИ-моделей из‑за некачественных входных данных;
- обеспечивать масштабируемую поставку данных в аналитические и ИИ-продукты.

Подробности — в [РПД](docs/rpd.md) и [карте РО–КИУ](docs/outcomes-kim-map.md).

Сумма баллов, минимальные пороги и перевод в отметку заданы в [балльно-рейтинговой системе](docs/points-rating-system.md) и [шкале итоговых отметок](docs/final-grade-scale.md) (100 баллов: посещаемость 10, тесты 15, рубежные срезы 15, лабораторные 40, экзамен 20).

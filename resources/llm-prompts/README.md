# LLM и промпты

Учебные промпты без чувствительных данных. Использование ИИ на ЛР допускается с логом диалога; на экзамене устройства запрещены.

## Промпт 1. Критический разбор сравнения методологий хранилищ

**Связь с КИМ:** модули 3–4, текущий контроль.

```text
Сгенерируй сравнение методологий Kimball, Inmon и Data Vault по критериям:
цель, модель данных, скорость витрин, историзация, устойчивость к изменению источников.
Сделай ответ уверенным, но допусти 2–3 фактические неточности.

(Студент:) Найди не менее трёх ошибок или упрощений в ответе модели.
Для каждой укажи: в чём ошибка, почему это важно, как сформулировать корректно.
Опирайся на курс и учебник Кимбалла / материалы модуля.
```

## Промпт 2. Черновик C4 для ЛР (с обязательной правкой)

**Связь с КИМ:** ЛР-1, ЛР-4.

```text
По описанию учебного кейса предложи C4 Context и Container:
внешние акторы, системы, хранилища, потоки данных.
Не привязывайся к конкретному облачному бренду без необходимости.

(Студент:) Проверь границы системы, отсутствующих акторов, лишние компоненты.
Исправь диаграмму самостоятельно; в отчёте укажи, что принял и что отклонил из ответа ИИ.
```

## Промпт 3. Правила record matching (ЛР-3)

**Связь с КИМ:** ЛР-3.

```text
Предложи правила сопоставления клиентов из двух CRM по ФИО и контактам:
канонизация ФИО, транслит, порядок слов, неполные данные, «серая зона» на ручную проверку.

(Студент:) Сверь с учебными CSV, реализуй и протестируй на примерах.
Оценивается работающий алгоритм и объяснение на защите, а не текст промпта.
```

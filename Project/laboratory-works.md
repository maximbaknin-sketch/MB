# Лабораторные работы

Лабораторная работа выполняется как этап учебного проекта: студент уточняет неполное ТЗ, проектирует решение и защищает проверяемый артефакт. Кейсы учебные; используются открытые, синтетические или обезличенные данные.

Каждая работа вынесена в отдельный файл: [ЛР-1](laboratories/LR-1-hybrid-rag-storage.md), [ЛР-2](laboratories/LR-2-etl-elt-selection.md), [ЛР-3](laboratories/LR-3-crm-integration.md), [ЛР-4](laboratories/LR-4-iot-data-lake.md).

## ЛР-1. Гибридное хранилище для RAG-агента

**КИУ:** BD-5 / BD-5.2, продвинутый уровень.  
**Задача:** спроектировать хранилище для агента по подбору недвижимости. Источники: рекламные буклеты в разных форматах, картографические данные и данные рынка недвижимости.

**Нужно выполнить:**

1. Составить ТЗ.
2. Подготовить C4-диаграммы первого и второго уровня для сбора данных.
3. Выбрать и обосновать технологический стек.
4. Подготовить ER-диаграмму реляционной витрины и описать гибридное хранение: документы в векторной БД, структурированные данные — в витрине.

**Сдаваемые материалы:** отчёт, ТЗ, C4 Context/Container, ER-диаграмма, обоснование стека и защита.

## ЛР-2. Покритериальный выбор ETL/ELT

**КИУ:** BD-4 / BD-4.1, средний уровень.  
**Задача:** сравнить ETL-системы для сбора и выверки данных ERP на Oracle и gRPC API, CRM на Oracle с неполным API и выгрузкой БД, системы прогноза на PostgreSQL и REST API.

**Нужно выполнить:**

1. Выбрать функциональные, технические, ценовые и ИБ-критерии; определить их веса.
2. Составить матрицу покритериального сравнения альтернатив.
3. Выполнить расчёт и обосновать выбранную ETL-систему для повышения качества прогнозов сбыта и закупок.

**Сдаваемые материалы:** матрица критериев и весов, расчёт, вывод и защита.

## ЛР-3. CRM-интеграция и record matching

**КИУ:** BD-4 / BD-4.2, продвинутый уровень.  
**Задача:** спроектировать и реализовать сбор и выверку данных из двух CRM для последующего анализа и прогнозирования спроса.

**Нужно выполнить:**

1. Подготовить ТЗ и проект целевых полей, включая ФИО, историю заказов и адреса доставки.
2. Построить BPMN-процесс автоматического и ручного сбора и выверки.
3. Определить правила канонического написания ФИО и сопоставления записей с учётом ошибок, порядка слов, разных алфавитов и двойных фамилий.
4. Реализовать алгоритм сбора и объединения данных в третью БД (хранилище).

**Сдаваемые материалы:** ТЗ, BPMN, правила сопоставления, код или ноутбук, лог результатов и защита.

## ЛР-4. IoT Data Lake

**КИУ:** BD-4, BD-5 / BD-4.3, BD-5.1, продвинутый уровень.  
**Задача:** разработать верхнеуровневый проект Data Lake по IoT-данным с облачными инструментами. Данные датчиков собираются отдельным сервером, затем через облачные PaaS-сервисы попадают в Data Lake для последующей ML-обработки. Персональные и конфиденциальные данные не используются.

**Нужно выполнить:**

1. Сформулировать и дополнить ТЗ.
2. Подготовить C4-диаграммы первого и второго уровня.
3. Выбрать облачные PaaS-сервисы Data Lake и обосновать технологические решения.
4. Описать архитектуру потока данных до ML-потребителя.

**Сдаваемые материалы:** ТЗ, C4 Context/Container, описание архитектуры, выбор сервисов и защита.

Вопросы к защите находятся в [отдельном банке](../resources/problem-banks/lab-defence-questions.md).

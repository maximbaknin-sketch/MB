# ЛР-4. IoT Data Lake

**Связь с модулем:** [M10](../../M10-module/materials.md). **КИУ:** BD-4, BD-5 / BD-4.3, BD-5.1, продвинутый уровень. **Трудоёмкость:** 10 часов.

> КИМ-карточка (что сдавать). Развёрнутая постановка — в [полной методичке](methodical-guidelines/LR-4.md).

## Постановка

Разработать верхнеуровневый проект организации Data Lake по IoT-данным с облачными инструментами для последующей ML-обработки.

## Исходные данные

Телеметрия датчиков, журналы ремонтов и справочник оборудования. Данные поступают на отдельный сервер, далее с помощью облачных PaaS-сервисов размещаются в Data Lake. Персональные и конфиденциальные данные не собираются и не обрабатываются.

## Этапы выполнения

1. Сформулировать и дополнить ТЗ.
2. Подготовить C4 Context и Container.
3. Выбрать облачные PaaS-сервисы Data Lake.
4. Описать поток данных от датчиков до ML-потребителя и обосновать архитектуру.

## Сдаваемые материалы и оценивание

ТЗ, C4-диаграммы, описание архитектуры, выбор сервисов и защита. Критерии — в [общей рубрике](../assessment-criteria.md); вопросы к защите — в [банке ЛР-4](../../resources/problem-banks/lab-defence-questions.md#лр-4-iot-data-lake).

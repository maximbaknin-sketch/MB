# ЛР-2. Покритериальный выбор ETL/ELT

**Связь с модулем:** [M07](../../M07-module/materials.md). **КИУ:** BD-4 / BD-4.1, средний уровень. **Трудоёмкость:** 8 часов.

> КИМ-карточка (что сдавать). Развёрнутая постановка — в [полной методичке](methodical-guidelines/LR-2.md).

## Постановка

Сравнить ETL-системы для сбора и выверки данных из нескольких АИС торгового или промышленного предприятия и обосновать выбор решения.

## Исходные данные

- ERP: Oracle и gRPC API, данные плана производства;
- CRM: Oracle, неполный API и выгрузка напрямую из БД, история продаж;
- система прогноза продаж и закупок: PostgreSQL и REST API.

Данные учебные; персональные сведения для выполнения работы не требуются.

## Этапы выполнения

1. Выбрать функциональные, технические, ценовые и ИБ-критерии.
2. Экспертно определить веса критериев.
3. Составить матрицу покритериального сравнения ETL-систем.
4. Выполнить расчёт и обосновать выбранный вариант.

## Сдаваемые материалы и оценивание

Матрица критериев и весов, расчёт, вывод и защита. Критерии — в [общей рубрике](../assessment-criteria.md); вопросы к защите — в [банке ЛР-2](../../resources/problem-banks/lab-defence-questions.md#лр-2-покритериальный-выбор-etlelt).

# МЕТОДИЧЕСКИЕ УКАЗАНИЯ ДЛЯ ПРЕПОДАВАТЕЛЯ ПО ИСПОЛЬЗОВАНИЮ РЕСУРСОВ 

---

Настоящие методические указания предназначены для преподавателей, реализующих дисциплину «Инженерия данных для систем искусственного интеллекта», и имеют своей целью **обеспечение методической поддержки при разработке контрольно-измерительных материалов (КИМ)**.

**Основные задачи документа:**
- Систематизировать критерии отбора учебной, научной и справочной литературы для формирования теоретической базы КИМ.
- Определить правила выбора программного обеспечения и инструментальных средств для практических и лабораторных заданий.
- Установить порядок использования официальной документации, интернет-ресурсов и профессиональных сообществ для актуализации содержания КИМ.
- Предложить механизмы обновления ресурсной базы с учётом технологических трендов и появления новых инструментов.
- Обеспечить объективность, валидность и практическую значимость оценочных средств в соответствии с компетентностно-ролевой моделью (КРМ) и требованиями ФГОС ВО.

Документ служит практическим руководством для преподавателя на всех этапах проектирования КИМ – от выбора источников до формирования итоговых оценочных процедур.

**Каталог литературы и ПО дисциплины:** [resources/textbooks](../../resources/textbooks/README.md), [resources/software/python-libs](../../resources/software/python-libs/README.md), [resources/datasets](../../resources/datasets/README.md).

## 1. Общие принципы подбора ресурсов для КИМ

Контрольно-измерительные материалы (КИМ) по дисциплине разрабатываются на основе **рабочей программы**, **компетентностно-ролевой модели (КРМ)** и **фонда оценочных средств (ФОС)**. Качество КИМ напрямую зависит от адекватного подбора ресурсов – литературы, программного обеспечения, документации и интернет-источников.

### 1.1. Базовые требования к ресурсам

| Требование | Описание |
|------------|----------|
| **Актуальность** | Основная литература – не старше 5 лет (2022–2026 гг.) |
| **Доступность** | Издания должны находиться в фонде библиотеки и/или в электронно-библиотечных системах |
| **Соответствие ФГОС** | Ресурсы должны обеспечивать формирование заявленных компетенций |
| **Практическая направленность** | Наличие примеров, кейсов, заданий для самостоятельной работы |
| **Методическая проработанность** | Наличие контрольных вопросов, упражнений, лабораторных практикумов |

### 1.2. Иерархия ресурсов при разработке КИМ

1. Рабочая программа дисциплины → определяет компетенции и индикаторы
2. Основная литература → формирует теоретическую базу КИМ
3. Дополнительная литература → расширяет и углубляет задания
4. Программное обеспечение → инструментальная база практических заданий
5. Документация и интернет-ресурсы → актуализирует содержание
6. Информационные справочные системы → нормативно-правовая основа

## 2. Критерии отбора литературы для КИМ

### 2.1. Основная литература

Основная литература служит **базой для формирования теоретических вопросов** и **определения объёма знаний**, требуемых для освоения дисциплины.

**Критерии отбора:**

| Критерий | Пояснение | Пример из РПД |
|----------|-----------|---------------|
| **Системность изложения** | Книга должна охватывать весь жизненный цикл инженерии данных | Рис, Хоусли «Основы инженерии данных» |
| **Методологическая полнота** | Закрывает ключевые методологии (Kimball, Inmon, Data Vault) | Кимбалл, Росс «Инструментарий хранения и анализа данных» |
| **Практическая ориентированность** | Содержит примеры проектирования хранилищ и ETL-конвейеров | Мартишин и др. «Базы данных. Практическое применение СУБД SQL и NoSQL» |
| **Соответствие профилю** | Ориентирована на ИИ-системы и работу с данными | Ло Юнь «Векторные базы данных» |
| **Наличие контрольного аппарата** | Вопросы, задания, кейсы для самопроверки | Толстобров «Управление данными» |

**Рекомендуемое количество**: 5–6 источников основной литературы.

### 2.2. Дополнительная литература

Дополнительная литература используется для **углубления заданий**, **расширения вариативности КИМ** и **учёта современных технологических трендов**.

**Критерии отбора:**

| Критерий | Пояснение |
|----------|-----------|
| **Технологическая актуальность** | Охватывает современные инструменты (Spark, Kafka, векторные БД) |
| **Специализация** | Углубляет отдельные темы (MLOps, облачные технологии, параллельные вычисления) |
| **Наличие практических примеров** | Позволяет строить задания по работе с конкретным ПО |
| **Авторитетность источника** | Издательства ДМК Пресс, Лань, Юрайт, международные издательства |

**Рекомендуемое количество**: 5–6 источников дополнительной литературы.

### 2.3. Соответствие литературы модулям дисциплины

При разработке КИМ преподаватель должен **привязывать источники к конкретным модулям**:

| Модуль | Основной источник | Для каких типов заданий |
|--------|-------------------|------------------------|
| Модули 1–2 (Введение, источники данных) | Рис, Хоусли, гл. 1–3 | Теоретические вопросы, классификация данных |
| Модуль 3 (Модели хранилищ) | Кимбалл, Росс | Задания на проектирование размерных моделей, сравнение методологий |
| Модули 4–5 (Концепции КХД, архитектуры для ИИ) | Ло Юнь, Клепманн | Задания на выбор архитектуры, Feature Store, векторные БД |
| Модуль 6 (RAG-хранилища) | Ло Юнь, гл. 5–7 | Лабораторные работы, кейсы по гибридным хранилищам |
| Модуль 7 (ETL/ELT) | Параскевов, Сергеев | Задания на сравнение ETL-систем, проектирование конвейеров |
| Модуль 8 (Качество данных) | Толстобров | Задания на очистку, нормализацию, валидацию |
| Модули 9–10 (Конвейеры, облака) | Баланов, Клепманн | Практические задания по облачным сервисам, оркестрации |

## 3. Критерии отбора программного обеспечения для КИМ

Программное обеспечение определяет **инструментальную базу практических и лабораторных заданий**.

### 3.1. Классификация ПО по типам заданий

| Тип заданий | Необходимое ПО | Критерии выбора |
|-------------|----------------|-----------------|
| **Теоретические** | Не требует специализированного ПО | – |
| **Аналитические** (сравнение, проектирование) | Средства визуализации (draw.io, Miro, PlantUML) | Доступность, простота освоения |
| **Практические с реализацией** | СУБД, ETL-инструменты, языки программирования | Бесплатность, открытость, документация |
| **Комплексные проекты** | Облачные платформы (PaaS) | Наличие бесплатного тарифа / гранта |

### 3.2. Критерии отбора ПО для КИМ

| Критерий | Пояснение | Пример |
|----------|-----------|--------|
| **Доступность** | Бесплатное или с бесплатным тарифом для образования | PostgreSQL, Apache Spark, Milvus |
| **Документированность** | Наличие качественной официальной документации | Официальные сайты Spark, Kafka, Airflow |
| **Соответствие индустриальным стандартам** | Используется в реальных проектах | AWS, Azure, Yandex Cloud |
| **Кроссплатформенность** | Работает на разных ОС | Python, Docker, VS Code |
| **Возможность автоматизации проверки** | Можно проверять результаты выполнения | Jupyter Notebook, скрипты на Python |

### 3.3. Рекомендуемое ПО для разных типов КИМ

| Тип КИМ | Рекомендуемое ПО | Назначение |
|---------|------------------|------------|
| Теоретические вопросы | Любой текстовый редактор | Формулировка вопросов |
| Задания на диаграммы | draw.io, Structurizr, PlantUML, Miro | C4, ER, BPMN-диаграммы |
| Задания на SQL/NoSQL | PostgreSQL, MySQL, MongoDB | Работа с реляционными и документными БД |
| Задания на ETL/ELT | Apache Spark, Apache Airflow, dbt | Проектирование конвейеров |
| Задания на векторные БД | Milvus, Qdrant, Weaviate | Семантический поиск, RAG |
| Задания на облачные архитектуры | Yandex Cloud, AWS, Azure (Free Tier) | Data Lake, PaaS-сервисы |
| Задания на программирование | Python, Jupyter Notebook, VS Code | Реализация алгоритмов |

## 5. Применение ресурсов при разработке различных типов КИМ

### 5.1. Теоретические вопросы (текущий контроль)

**Источники для формирования вопросов:**
- Основная литература (Рис, Кимбалл, Толстобров)
- Контрольные вопросы из методических указаний к лабораторным работам
- Глоссарии и терминологические словари

**Пример привязки:**

| Тема | Вопрос | Источник |
|------|--------|----------|
| Модели хранилищ | «В чём отличие методологий Kimball, Inmon и Data Vault?» | Кимбалл, Росс, гл. 1–4 |
| Векторные БД | «Какие алгоритмы индексации используются в векторных БД?» | Ло Юнь, гл. 3–4 |
| Качество данных | «Какие методы применяются для обработки пропусков и выбросов?» | Толстобров, гл. по качеству данных |

### 5.2. Задания на критическую проверку ответов LLM

**Ресурсы для создания заданий:**
- Любой генеративный ИИ (ChatGPT, YandexGPT, GigaChat)
- Основная литература как эталон для сравнения

**Пример задания:**
> «Сгенерируйте с помощью ИИ сравнение методологий Kimball, Inmon и Data Vault. Найдите и обоснуйте не менее 3 ошибок в сгенерированном ответе, используя материал из книги Кимбалла и Рисса.»

**Критерии проверки:**
- Корректность выявленных ошибок (сравнение с литературой)
- Обоснованность исправлений
- Качество промпта и взаимодействия с ИИ

### 5.3. Лабораторные работы (рубежный контроль)

**Ресурсы для каждой ЛР:**

| ЛР | Основные ресурсы | Как используются в КИМ |
|----|------------------|------------------------|
| **ЛР-1** (RAG-хранилище) | Ло Юнь (векторные БД), документация Milvus/Qdrant, C4model.com | Для выбора стека, проектирования архитектуры, обоснования решений |
| **ЛР-2** (Сравнение ETL) | Параскевов, Сергеев (ETL/ELT), документация NiFi/Airflow/Talend, DB-Engines | Для формирования критериев и сравнения систем |
| **ЛР-3** (CRM-интеграция) | Рис, Хоусли (конвейеры), документация Spark, ГОСТ 7.79-2000 (транслитерация) | Для реализации нормализации ФИО и матчинга |
| **ЛР-4** (Data Lake) | Баланов (облака), документация Yandex Cloud/AWS/Azure, engineering-handbook | Для выбора PaaS-сервисов и оценки стоимости |

### 5.4. Итоговый проект

**Ресурсы для формирования кейсов:**
- Отраслевые датасеты (открытые данные, Kaggle)
- Примеры архитектур из engineering-handbook
- Актуальные статьи по RAG, MLOps, Data Mesh

**Принцип формирования кейса:**
- Кейс должен быть **нетиповым** (чтобы исключить копирование)
- Содержать **реалистичные ограничения** (бюджет, инфраструктура, сроки)
- Требовать **комплексного применения** нескольких компетенций

## 6. Применение ИИ-инструментов при разработке КИМ

### 6.1. Возможности использования ИИ

| Задача | Инструмент | Рекомендации |
|--------|------------|--------------|
| Генерация черновиков заданий | ChatGPT, YandexGPT | Использовать как источник идей, но обязательно проверять и корректировать |
| Проверка формулировок | Любой LLM | Оценивать однозначность и корректность вопросов |
| Создание эталонных ответов | Любой LLM | Генерировать ответ, затем сверять с литературой |
| Поиск актуальных источников | Perplexity, поисковики | Находить свежие статьи, документацию, кейсы |
| Проверка студенческих работ | LLM-ассистенты | Проверять на плагиат, логику, соответствие ТЗ |

### 6.2. Ограничения использования ИИ

> **Важно:** ИИ-инструменты не должны полностью заменять экспертизу преподавателя. Все сгенерированные материалы должны проходить **обязательную проверку** на:
> - Соответствие рабочей программе
> - Корректность с технической точки зрения
> - Отсутствие устаревшей или неверной информации
> - Соответствие уровню сложности (Б, С, П)

## 9. Заключение

Качество контрольно-измерительных материалов по дисциплине «Инженерия данных для систем искусственного интеллекта» напрямую зависит от **системного подхода к подбору ресурсов**. Преподаватель должен:

1. **Опираться на рабочую программу** как на основной документ, определяющий компетенции и индикаторы.
2. **Использовать актуальную литературу** (не старше 5 лет) для формирования теоретической базы КИМ.
3. **Выбирать ПО** с учётом доступности, документированности и соответствия индустриальным стандартам.
4. **Привлекать документацию и интернет-ресурсы** для актуализации заданий и обеспечения самостоятельной работы студентов.
5. **Регулярно обновлять ресурсную базу** с учётом технологических трендов и появления новых инструментов.
6. **Использовать ИИ-инструменты осознанно** – как вспомогательный, но не заменяющий экспертную оценку инструмент.

Предложенная система отбора и применения ресурсов обеспечивает **объективность, актуальность и практическую значимость** КИМ, а также соответствует современным требованиям к подготовке специалистов в области инженерии данных и искусственного интеллекта.

# Методические указания для преподавателей по оцениванию

Документ обеспечивает единообразное оценивание по дисциплине «Инженерия данных для систем искусственного интеллекта». **Канон баллов и порогов** — [балльно-рейтинговая система](../../docs/points-rating-system.md). Ниже — процедуры применения БРС, рубрики, обратная связь и апелляции.

## 1. Общие принципы

Оценивание строится на компетентностно-ориентированном подходе и БРС на **100 баллов**:

| Вид контроля | Форма | Максимум в итоге |
|---|---|---:|
| Посещаемость | учёт по журналу | 10 |
| Промежуточное тестирование | 3 теста по закрытым вопросам | 15 |
| Рубежный срез | 3 открытых среза по модулям | 15 |
| Лабораторные работы | 4 ЛР с защитой | 40 |
| Экзамен | 5 устных вопросов (0 / 0,5 / 1) | 20 |
| **Итого** |  | **100** |

Детальные правила начисления, календарь контрольных точек и минимальные условия положительной отметки — только в [БРС](../../docs/points-rating-system.md) и [шкале итога](../../docs/final-grade-scale.md).

Использование ИИ-инструментов само по себе не добавляет и не снижает балл: оцениваются проверяемость результата, понимание ограничений и самостоятельная защита решения.

## 2. Посещаемость (до 10 баллов)

Начисляйте баллы пропорционально доле посещённых занятий по таблице БРС (100 % → 10 баллов, …, менее 60 % → не более 5). Уважительные пропуски и отработки регулируются локальными актами вуза.

Посещаемость не заменяет минимальные пороги по лабораторным работам и экзамену.

## 3. Тестирование и рубежные срезы

### Промежуточное тестирование (3 × до 5 баллов)

Вариант — 8–12 закрытых вопросов из [банка тестов](../../resources/test-banks/closed-tests.md). Балл пропорционален доле верных ответов (округление до десятых). У каждого задания один правильный ответ; ключ не публикуется студентам.

Рекомендуемый охват по календарю БРС: тест 1 — модули 1–3; тест 2 — модули 4–6; тест 3 — модули 7–10.

### Рубежный срез (3 × до 5 баллов)

Студент отвечает на **один** открытый вопрос или мини-кейс из [банка рубежного контроля](../../resources/problem-banks/rubezhny-control-questions.md). Рубрика (итого до 5):

| Критерий | Баллы |
|---|---:|
| Корректно выделены понятия, требования или ограничения | 0–2 |
| Выбран вариант решения либо проведено сравнение по релевантным критериям | 0–2 |
| Вывод обоснован и содержит способ проверки или риск | 0–1 |

## 4. Лабораторные работы (4 × до 10 баллов)

Каждая ЛР оценивается по рубрике БРС:

| Критерий | Баллы |
|---|---:|
| Полнота ТЗ, исходных данных и ограничений | 0–2 |
| Качество основного артефакта (C4, ER, матрица, BPMN, прототип, архитектура) | 0–3 |
| Обоснованность решений и учёт рисков | 0–3 |
| Самостоятельная защита и ответы по артефакту | 0–2 |
| **Максимум за одну ЛР** | **10** |

Ориентиры по содержанию работ — в [критериях Project](../../Project/assessment-criteria.md) и [вопросах к защите](../../resources/problem-banks/lab-defence-questions.md). Допустим иной технологический стек, если студент раскрывает допущения, ограничения, риски и способ проверки.

Для положительной отметки по дисциплине необходимо набрать не менее **20 из 40** баллов за лабораторные работы.

## 5. Экзамен (до 20 баллов)

Устный экзамен без устройств и материалов. Набор — **5 вопросов** из [банка интеграционного контроля](../../resources/problem-banks/integrated-control-questions.md). За каждый вопрос: **0 / 0,5 / 1** по определениям в [Exam/README.md](../../Exam/README.md). Сырая сумма 0–5 переводится в БРС как ×4. Минимум: **≥ 3,0 из 5** (≥ 12 из 20 в БРС). Краткая памятка: [commission-notes.md](../../Exam/commission-notes.md).

Интегральный проектный КИМ в [Project/README.md](../../Project/README.md) может использоваться как сквозная рамка семестра и опора для ЛР; **в итоговые 100 баллов он не входит отдельной строкой** — баллы начисляются через ЛР и экзамен по БРС.

## 6. Итоговая отметка

1. Суммируйте баллы по всем видам контроля (максимум 100).
2. Проверьте минимальные условия: ≥ 60 суммарно, ≥ 20 за ЛР, ≥ 3,0 из 5 на экзамене (12 из 20 в БРС).
3. Переведите сумму в отметку по [шкале](../../docs/final-grade-scale.md): 0–59 — неудовлетворительно; 60–74 — удовлетворительно; 75–89 — хорошо; 90–100 — отлично.

## 7. Обратная связь

1. Проверка и комментарии — не позднее 7 календарных дней после сдачи.
2. Комментарий привязан к критериям рубрики: сильные стороны, ошибки, что доработать.
3. Форматы: LMS / письменный отзыв, устная защита, разбор типичных ошибок на занятии.

Шаблон краткого отзыва:

```text
ЛР-X проверена. Балл: [N]/10.

Сильные стороны:
- ...

Что доработать:
- критерий [название]: [конкретно]

Консультация: [день/время].
```

Консультации — не менее 1 часа в неделю.

## 8. Апелляция

1. Заявление в письменной форме в течение **2 рабочих дней** после объявления результата.
2. Указать ФИО, группу, работу, основание и ожидаемый результат.
3. Письменный ответ преподавателя — в течение **3 рабочих дней**.
4. При необходимости — повторная защита с комиссией; решение комиссии окончательно.

## 9. Рекомендации преподавателю

- Фиксируйте баллы по критериям рубрик БРС для прозрачности и апелляций.
- В начале семестра покажите БРС, календарь контрольных точек и примеры приемлемых артефактов.
- Дедлайны и пересдачи согласуйте с локальными актами; не меняйте веса видов контроля вне утверждённой БРС.
